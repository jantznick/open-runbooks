# Application security program — Phase guides

Each phase document (`01`–`06`) uses the **same structure** so teams and documentation portals can navigate predictably.

| Section | Purpose |
|---------|---------|
| **Policy alignment** | Which baseline controls this phase satisfies |
| **Navigation** | Links to program overview and adjacent phases |
| **Objective** | One-paragraph goal for the phase |
| **OWASP SAMM mapping** | Primary/secondary SAMM functions and practices |
| **Scope** | What work items belong in this phase |
| **Core activities** | What teams do day to day |
| **Reference controls** | Runnable jobs, templates, or roadmap status for this phase |
| **Suggested tooling** | Tool-agnostic and corporate reference implementations |
| **Minimum viable process** | Smallest compliant operating pattern |
| **Common pitfalls** | Frequent failures to avoid |
| **Inputs** / **Outputs** | Artifacts in and out of the phase |
| **Success metrics** | How to measure effectiveness |
| **Adoption paths** | Minimum viable vs scaled paths, tooling-by-use-case table, 30/60/90 rollout |
